## 대규모 프로젝트

### 1. W컨셉 전사 사이트 리뉴얼 (2024)

**개요**

- 전체 서비스 UI/UX 개선 및 디자인 전면 개편
- 숏폼 콘텐츠 도입을 통한 새로운 쇼핑 경험 제공
- 메인 페이지 및 주요 카테고리 페이지 리뉴얼

**배경**

- 숏폼 트렌드 반영 및 콘텐츠 커머스 강화
- 변화하는 사용자 니즈 및 디자인 트렌드 반영
- 모바일 사용자 증가에 따른 UX 최적화 필요

**주요 작업**

- 메인: 퀵메뉴, WDNA, 프리쇼, 테마 섹션 마크업
- 숏폼 콘텐츠 영역 신규 구현
- 한정 특가 페이지
- 신상/세일/추천 카테고리 리뉴얼
- 오리지널 콘텐츠 페이지
- 스타일클립 등록 시스템

**기술적 성과**

- 기 구축된 모노레포 구조 기반 개발
- CSS Grid, Flexbox 기반 반응형 레이아웃 구축
- Intersection Observer를 활용한 이미지 지연 로딩 구현
- 숏폼 비디오 플레이어 최적화
- BEM 방법론 적용으로 CSS 유지보수성 개선
- 웹 접근성 KWCAG 2.1 AA 준수
- Chrome/Safari/Firefox/Edge 크로스 브라우징 대응

### 2. 프로모션 페이지 빌더 시스템 구축 (2022-2025)

**개요**

- 비개발자가 직접 이벤트 페이지를 제작할 수 있는 컴포넌트 기반 CMS
- 기획전, 메가프로모션, 더블유위크 등 다양한 프로모션 대응

**배경**

- 매주 5-10개 기획전 페이지 제작 반복 작업
- 일관된 UI/UX 품질 유지 필요

**주요 작업**

- 20여개 재사용 가능한 프로모션 컴포넌트 개발
- 더블유위크 전용 컴포넌트 신규 개발
- 특가/내비게이션 컴포넌트 개선
- 메가프로모션 코너 신설

**기술적 성과**

- Component 기반 모듈 아키텍처 설계
- SCSS 변수 시스템으로 테마 관리
- BEM 네이밍 컨벤션 적용
- CSS Grid 기반 유연한 레이아웃 구현
- 반응형 breakpoint 전략 수립

## 기술 스펙

### 개발 환경

- **Frontend**: React, TypeScript
- **Styling**: Styled-components
- **UI Library**: Swiper, Custom Components
- **Package Manager**: Yarn (Workspace)

### 웹 표준 & 접근성

- WCAG 2.1 Level AA 준수
- 시맨틱 마크업
- 키보드 네비게이션 지원
- ARIA 속성 활용
- 명도 대비 4.5:1 이상

### 반응형 대응

- Mobile (~ 767px)
- Tablet (768px ~ 1023px)
- Desktop (1024px ~)
- Mobile-first 접근

### 크로스 브라우징

- Chrome, Safari, Firefox, Edge 최신 2개 버전
- iOS Safari 14+
- Android Chrome 최신 버전

### 성능 최적화

- 이미지 지연 로딩 (Intersection Observer)
- 코드 스플리팅
- Swiper 라이브러리 활용

### 코드 품질

- ESLint + TypeScript
- BEM 방법론
- Component 기반 설계
- Git 브랜치 전략
